# Design

This folder contains the **product/design-level** artifacts for Play-by-Play Admin: use cases, requirements, architecture views, domain language, and early prototype references.

If you want **operator documentation** (screenshots and click-path workflows), use `Documentation/User.md` (see `Documentation/README.md`).

## What each file is for

- **`UseCases.md`**: Actor list + end-to-end scenarios (UC1+) mapped to functional requirements.
- **`Requirements.md`**: Functional requirements (FR*) and non-functional requirements (NR*).
- **`BusinessRequirements.md`**: Business-level intent (“why”) behind the admin platform.
- **`Architecture.md`**: High-level system/module architecture (presentation vs business vs data).
- **`DomainModel.md`**: Core domain concepts and a lightweight class diagram for shared vocabulary.
- **`TechStack.md`**: Major technology choices and rationale.
- **`Prototype.md`**: Early interactive prototype links/notes (historical context; the production product may differ).
